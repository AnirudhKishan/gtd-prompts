# GTDv2 --- Weekly Review Process

Operationalizes: "GTDv2 - Philosophies, Rationales and Principles.md"

Goal: maintain a trustworthy attention system with minimal cognitive
load.

------------------------------------------------------------------------

# Lists

    In
    Active Projects
    Maybe
    Cold Storage

## Meaning of Lists

**In** → capture bucket, unclarified inputs

**Active Projects** → projects you are currently willing to carry
mentally. WIP‑limited. Only active projects may have next actions.

**Maybe** → everything you might do someday. Single list, no distance
judgment. Not capped — when it feels noisy, push stale items to Cold
Storage.

**Cold Storage** → ideas you don't want to delete but don't want to see.
Archive. Not capped.

------------------------------------------------------------------------

# Key Design Decisions

## 1. Single Commitment Boundary

Only **Active Projects** implies commitment.

Crossing this boundary means:

-   project consumes WIP capacity
-   project may receive next actions

Invariant:

> No Active Project → No Next Action

All other lists are safe parking zones.

------------------------------------------------------------------------

## 2. One Maybe List, Not Many

Previous versions used multiple horizon lists (Review Cycle / Month /
Year / Someday) to manage attention distance. This created graduated
classification decisions on every item during review — the primary
source of friction.

**Cold Storage** — not more horizons — is the right noise filter.
When Maybe feels long, push items you don't want to see to Cold Storage.
This is a binary "do I want to keep seeing this?" decision, not a
distance judgment.

------------------------------------------------------------------------

## 3. Local Decisions During Review

Each step asks one simple question. Do not compare across lists.

------------------------------------------------------------------------

## 4. Activation Decisions Are Isolated

Promotion to **Active Projects** happens only in Step 4.

Purpose:

-   prevent earlier steps from becoming global prioritization
-   keep most review decisions local and mechanical

------------------------------------------------------------------------

## 5. Dead Items Are Deleted

Dead means deleted from the system entirely.

Cold Storage is for items that are **not dead** but that you don't want
to see. Items that are truly done, cancelled, or irrelevant are removed.

------------------------------------------------------------------------

# Weekly Review

Steps 1 and 2 are essential.\
Steps 3, 4, and 5 are skippable.

If In is empty and Active feels right, the review can be two minutes.

------------------------------------------------------------------------

## Step 1 --- Empty In

Goal: empty capture.

For each item in In, ask: **"What is this?"**

Trash → delete\
Reference → file it\
Everything else → **Maybe**

**Exception:** if this is something you are already actively doing right
now and Active has room (< 10), place it directly into Active Projects.
This should be rare — the default is always Maybe.

No next actions here. No deliberation. Just park it.

------------------------------------------------------------------------

## Step 2 --- Gut‑Check Active

Cap: Active Projects ≤ 10

For each active project, ask: **"Am I still carrying this?"**

Yes → keep\
No, but still alive → **Maybe**\
Dead → **delete from the system**

Don't deliberate. If you hesitate, keep it. You can drop it next week.

------------------------------------------------------------------------

## Step 3 --- Skim Maybe (skippable)

**Don't review every item.** Skim the list.

Three questions:

1.  "Anything here that's dead?" → **delete from the system**
2.  "Anything I don't want to see but it's not dead?" → **Cold Storage**
3.  "Anything here that's calling to me?" → **star / flag it** for
    Step 4

If nothing jumps out, move on.

------------------------------------------------------------------------

## Step 4 --- Activate? (skippable)

Only if you starred something in Step 3 **and** there is room in Active
(< 10):

> "Do I want to commit to this right now?"

Yes → move to Active Projects\
Not sure → leave it in Maybe. It will be there next week.

Activation should feel like **"finally"** not "maybe."

------------------------------------------------------------------------

## Step 5 --- Cold Storage Skim (quarterly, or whenever curiosity strikes)

There is no schedule for this step. It exists only if you want it.

> "Anything worth reviving?"

Move to Maybe if yes. Otherwise ignore.

------------------------------------------------------------------------

# Exit Ramps

-   Steps 1 and 2 are essential. Steps 3–5 are skippable.
-   If you only complete Steps 1 and 2, you have done a useful review.
-   Completion is optional. Safety is required.

------------------------------------------------------------------------

# WIP Limits

Active Projects ≤ 10 (hard cap)\
Maybe → soft target ~25 (push stale items to Cold Storage when noisy)\
Cold Storage → uncapped

------------------------------------------------------------------------

# Migration from Previous System

1.  Merge Review Cycle Maybe + Month Maybe + Year Maybe + Someday Maybe
    → single **Maybe** list
2.  Keep Active Projects as‑is
3.  Keep Cold Storage as‑is
4.  No re‑sorting needed — everything is just Maybe now
